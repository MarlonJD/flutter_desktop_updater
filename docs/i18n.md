# Internationalization

desktop_updater keeps i18n app-owned. The package does not register a Flutter
`LocalizationsDelegate`, ship ARB files, or choose a locale. Your app resolves
the active locale and passes translated copy to `DesktopUpdaterController`
through `DesktopUpdateLocalization`.

The README includes short localization snippets for release notes and update
failure tooltips. This guide shows a fuller app-level setup.

## Basic Wiring

Pass a `DesktopUpdateLocalization` when you create the controller:

```dart
final controller = DesktopUpdaterController(
  appArchiveUrl: Uri.parse("https://updates.example.com/app-archive.json"),
  localization: const DesktopUpdateLocalization(
    updateAvailableText: "Update available",
    newVersionAvailableText: "{} {} is available",
    newVersionLongText:
        "A new version is ready. This will download {} MB of data.",
    downloadText: "Download",
    restartText: "Restart to update",
  ),
);
```

The `{}` tokens are package-owned placeholders. The ready-made UI replaces them
later, after it knows the selected app name, version, or download size.

## Locale-Aware Controller Example

Most apps should adapt their existing Flutter i18n layer into
`DesktopUpdateLocalization`. The example below uses a small Dart adapter, but
the same shape works when the string values come from generated
`AppLocalizations`, a JSON catalog, or a remote configuration service.

```dart
import "dart:async";
import "dart:io";

import "package:desktop_updater/desktop_updater.dart";
import "package:desktop_updater/updater_controller.dart";
import "package:flutter/material.dart";

class UpdatesPanel extends StatefulWidget {
  const UpdatesPanel({
    super.key,
    required this.appArchiveUrl,
  });

  final Uri appArchiveUrl;

  @override
  State<UpdatesPanel> createState() => _UpdatesPanelState();
}

class _UpdatesPanelState extends State<UpdatesPanel> {
  DesktopUpdaterController? _controller;
  Locale _locale = const Locale("en");

  @override
  void didChangeDependencies() {
    super.didChangeDependencies();

    final nextLocale = Localizations.localeOf(context);
    final localeChanged = nextLocale != _locale;
    _locale = nextLocale;

    final localization = desktopUpdaterLocalizationFor(_locale);
    final controller = _controller;
    if (controller == null) {
      _controller = DesktopUpdaterController(
        appArchiveUrl: widget.appArchiveUrl,
        localization: localization,
        releaseNotesLoader: (descriptor) {
          return fetchLocalizedReleaseNotes(
            descriptor: descriptor,
            locale: _locale,
          );
        },
      );
      return;
    }

    controller.localization = localization;
    if (localeChanged &&
        controller.canLoadReleaseNotes &&
        controller.releaseNotesState is ReleaseNotesLoaded) {
      unawaited(controller.loadReleaseNotes(forceRefresh: true));
    }
  }

  @override
  void dispose() {
    _controller?.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    final controller = _controller;
    if (controller == null) {
      return const SizedBox.shrink();
    }

    return DesktopUpdateDirectCard(
      key: ValueKey(_locale.toString()),
      controller: controller,
    );
  }
}

Future<ReleaseNotes> fetchLocalizedReleaseNotes({
  required ReleaseDescriptor descriptor,
  required Locale locale,
}) async {
  final response = await myReleaseNotesApi.fetchJson(
    version: descriptor.version,
    platform: descriptor.platform,
    channel: descriptor.channel,
    locale: locale.toLanguageTag(),
  );
  return ReleaseNotes.fromJson(response);
}
```

The `ValueKey` forces the ready-made widget subtree to rebuild when the locale
changes. The `releaseNotesLoader` captures the current `_locale`, so localized
release notes requests use the latest language. If notes were already loaded,
`forceRefresh: true` refreshes the controller cache for the new locale.

## Full Localization Factory

Keep the package placeholders as literal `{}` tokens in translated templates.
The current replacement order is:

- `newVersionAvailableText`: app name, version
- `newVersionLongText`: download size in MB
- `supportPolicyWarningText`: minimum supported version, enforcement date
- `upToDateText`: app/version label

The support-policy date is formatted before it is inserted into `{}`. The
package default is `YYYY-MM-DD HH:mm UTC`; pass `formatDateTime` when an app
needs a locale-specific or local-time display.

```dart

        warningTitleText: "Guncelleme kurulsun mu?",
        restartWarningText:
            "Kurulumu tamamlamak icin uygulama yeniden baslatilacak.\n"
            "Kaydedilmemis degisiklikler kaybolabilir.",
        warningCancelText: "Simdi degil",
        warningConfirmText: "Yeniden baslat",
        saveFirstText: "Once kaydet",

        freshInstallRequiredText:
            "Bu surum guncellemeyi guvenli sekilde kuramaz. "
            "Lutfen en son surumu indirin.",

        supportPolicyBlockedText:
            "Bu surum artik desteklenmiyor. Devam etmek icin guncelleyin.",
        formatDateTime: (dateTime) {
          final local = dateTime.toLocal();

        onUpdateFailedTooltip: (error) {
          return localizedUpdateFailureTooltip(error, locale);
        },
        releaseNotesButtonTooltipText: "Release notes",
        releaseNotesTitleText: "What's new",
        releaseNotesTypeLabels: {
          "feat": "New features",
          "fix": "Bug fixes",
          "other": "Other changes",
        },
        releaseNotesSectionLabels: {
          "features": "New features",
          "fixes": "Bug fixes",

String? localizedUpdateFailureTooltip(Object error, Locale locale) {
  final isTurkish = locale.languageCode == "tr";
  if (error is SocketException) {
    return isTurkish ? "Internet baglantisi yok." : "No internet connection.";
  }
  if (error is TimeoutException) {
    return isTurkish
        ? "Guncelleme sunucusu zaman asimina ugradi."
        : "The update server timed out.";
  }
  return null;
}
```

If you use generated `AppLocalizations`, the factory can read static labels from
that object. For the three package-filled template fields, keep values as
literal `{}` templates because the package substitutes them after the active
update descriptor is known.

## Localized Release Notes

Use `releaseNotesLoader` for localized notes. It receives the selected
`ReleaseDescriptor`, and your closure can add the active locale before calling
your API:

```dart
final controller = DesktopUpdaterController(
  appArchiveUrl: archiveUrl,
  localization: desktopUpdaterLocalizationFor(locale),
  releaseNotesLoader: (descriptor) async {
    final json = await api.getReleaseNotes(
      version: descriptor.version,
      platform: descriptor.platform,
      channel: descriptor.channel,
      locale: locale.toLanguageTag(),
    );
    return ReleaseNotes.fromJson(json);
  },
);
```

For static hosting, publish separate locale files and choose the URL in your
loader:

```text
updates/
  releases/2.4.0/macos/release-notes.en.json
  releases/2.4.0/macos/release-notes.tr.json
```

The hosted JSON can include its locale for diagnostics or custom UI:

```json
{
  "schemaVersion": 1,
  "format": "desktop_updater.release_notes.v1",
  "locale": "tr",
  "summary": "Kalite ve guvenilirlik iyilestirmeleri.",
  "sections": [
    {
      "type": "features",
      "title": "Yeni ozellikler",
      "items": [
        { "body": "Guncelleme notlari artik uygulama diline gore yuklenir." }
      ]
    }
  ]
}
```

## Custom UI

`DesktopUpdateLocalization` only affects the bundled updater widgets and
helpers. If you build a fully custom update surface from `controller.state`,
use your app's normal localization layer directly and call controller methods
such as `checkForUpdates()`, `downloadUpdate()`, `restartApp()`, and
`loadReleaseNotes()`.
